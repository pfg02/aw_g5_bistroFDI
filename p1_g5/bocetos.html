Bocetos de la Aplicación

A continuación, presentamos los diseños iniciales de las distintas pantallas de la aplicación Bistro FDI.

  • Funcionalidad 0 - Gestión de Usuarios
  • Funcionalidad 1 - Gestión de Productos
  • Funciondlidad 2 - Gestión de Pedidos

Funcionalidad 0. Gestión de Usuarios

Descripción: Gestión login
Pantalla de autenticación donde el usuario introduce su nombre de usuario y contraseña para acceder a la aplicación.
Descripción: Iniciar y registro
Vista que permite iniciar sesión o registrar un nuevo usuario. En el registro se almacenan nombre de usuario único, email, nombre, apellidos y contraseña. El rol por defecto es cliente.
Descripción: Perfil
Pantalla donde el usuario puede visualizar y actualizar sus datos personales, incluyendo nombre de usuario, email, nombre, apellidos y rol asignado.
Descripción: Avatar
Permite seleccionar un avatar predeterminado, subir una imagen personalizada o eliminar la foto para usar el avatar por defecto.
Descripción: Asignar nuevo rol
Vista para que el gerente pueda modificar el rol de un usuario. Los roles son jerárquicos: cliente, camarero, cocinero y gerente.

Funcionalidad 1 - Gestión de Productos

Descripción: Listado de categorías
Vista principal donde el gerente puede visualizar todas las categorías existentes con su nombre, descripción e imagen.
Descripción: Crear nueva categoría
Formulario que permite añadir una nueva categoría introduciendo nombre, descripción e imagen opcional.
Descripción: Editar categoría
Permite modificar los datos de una categoría ya creada, incluyendo cambiar o eliminar su imagen.
Descripción: Detalle de categoría
Vista individual que muestra la información completa de una categoría.
Descripción: Gestión de productos
Vista principal de gestión de productos donde se muestran los ocho atributos: nombre, descripción, categoría, imagen, precio base, IVA, disponible y ofertado.
Descripción: Añadir producto
Formulario para crear un nuevo producto introduciendo todos sus atributos.
Descripción: Editar producto
Permite modificar la información del producto y actualizar automáticamente el precio final con IVA.
Descripción: Confirmación retirar producto
Ventana de confirmación para retirar un producto de la carta. Esta acción actualiza la propiedad "ofertado" sin eliminar el producto de la base de datos.

Funcionalidad 2 - Gestión de Pedidos

Descripción: Pantalla de inicio
El cliente se identifica y elige entre iniciar un nuevo pedido o consultar su historial.
Descripción: Tipo de pedido
El cliente elige si el pedido es para consumir en el local o para llevar.
Descripción: Carta
El cliente añade a su pedido productos de la carta (ordenados por categoría).
Descripción: Resumen del pedido
El cliente revisa su pedido y elige entre confirmarlo y pagar o cancelarlo.
Descripción: Realizar pago
El cliente elige entre introducir los datos de su tarjeta y realizar el pago o pagar en caja.
Descripción: Pedido confirmado
Pantalla de confirmación del pedido una vez realizado el pago.
Descripción: Lista de pedidos.
El cliente puede ver la lista de sus pedidos con el estado y un pequeño resumen.
Descripción: Gestión de pedidos (camarero).
El camarero puede ver los pedidos que hay pendientes, preparar el pedido para llevar y ponerlo como terminado, o llevarlo al cliente y ponerlo como entregado.
